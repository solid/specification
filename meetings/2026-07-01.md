### Proposal: intent handover (login as + open resource)
https://github.com/solid/specification/issues/799
* YB: I did a demo on this at SoSy.
* ... idea is that I have a desktop-y app with icons, and the idea is that I can click on an app icon and I would be automatically be logged in in that app I clicked.
* ...: Noel implemented the logic
* ...: JW was also looking into whether the file browser can faciliate that
* ...: JW also noted to maybe start a spec
* eP: Do you have a demo?
* YB: Yes! ... demo time ...
* ... in this server we have a default app, there is an autologin.
* ... I logged into my Solid Server - which does not have a proper app but a "bootloader" which looks into my Solid Profile where it found that my default app is this Solid Launcher that just opened (which I prefer).
* ... and the idea is then that I dont want to manually login but automatically
* ... So the intent to be logged in is transmitted automatically
* eP: is that an existing session?
* YB: No, these are fresh sessions
* eP: what if there is an unkown app that is redirected to?
* YB: then the user must consent to log in (and thus authorize the app one first time)
* eP: What about FedCM? It also reduces the friction on entering the WebID
* YB: I think I still need to tell each app who I want to log in as/with.
* ... what is missing then is that I can just take my existing identity directly into the app
* eP: maybe we can track whether or where there is overlap with FedCM
* eP: 1. How do you prompt the user
* eP: 2. How do you do delegation
* YB: On one, it is just a consent to check whether the app is trusted to get a token or not.
* YB: The app, when intended to login with your WebID, it goes to your IDP logs you in, and then you use it.
* ... it is basically just filling in the authentication stuff for you under the hood
* eP: So there is no extra authroization happening?
* YB: No, just the usual flow we have today. I am aiming for a most seamless experience to hopping from one app to another without interruption.
* ...: So, when you see a file in a file browser and want to edit that, just opens in the editor app, be automatically logged in there and use that, so not every app has to implement a file browser or editor ... so apps can re-use / rely on external functionality provided by other apps.
* CB: The idea is great, we also use something similar in MANDAT where buisness app would indicate which access request should be looked at. We didn't give WebID or identity provider.
* ...: I still need to look weater you need to exploint it somehow
* eP: In SAI we also use "resource indication" to signal to authorization agent which resoure user selected in application to share
